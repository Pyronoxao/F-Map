# F-Map Schema

## System

A system is a major part of a project.

Example:

- Authentication
- Database
- User Interface
- Payments

---

## Dependency

A dependency is something another system relies on.

Example:

Payments
  depends on
Users

---

## Change Impact

When one system changes:

F-Map must identify:

- What is affected
- What is not affected
- Estimated AI cost
- Risk level

---

## Build Flow

User Request
→ F-Map Analysis
→ Impact Report
→ User Approval
→ AI Execution
